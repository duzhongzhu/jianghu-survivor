# 江湖求生 (Jianghu Survivor) — 游戏设计规格

## 概述

**类型：** 俯视角波次生存动作游戏
**主题：** 像素武侠
**引擎：** Godot 4.x（GDScript）
**目标：** 一个完整可玩的「小游戏」，核心循环：配置装备 → 战斗 → 升级成长 → 死亡结算 → 强化再战

---

## 核心循环

```
配置装备技能 → 进入战斗 → 敌人波次递增
    ↑                              ↓
    └── 死亡结算（获得铜钱/修为）←── 升级选技能/属性
```

---

## 操控

- **WASD** — 移动
- **自动普攻** — 最近敌人自动攻击，攻击形态由武器决定
- **技能键 Q / E / R** — 3 个主动技能槽，释放在当前移动方向
- **无需瞄准** — 自动锁定最近敌人，方向性技能朝 WASD 方向释放

---

## 战斗系统

### 规则
- 在一片像素风江湖场景中，敌人从四方涌来
- 击杀敌人获得经验
- 经验条满 → 升级 → 弹出 3 选 1 面板，暂停战斗
- 每 5 波出现精英敌人，每 10 波出现 Boss
- 血量归零 → 战斗结束
- 波次间有 8 秒喘息间隔

### 升级选项（3 选 1）
三种类型混合，按稀有度加权随机：
1. **新技能** — 获得尚未拥有的主动/被动技能
2. **技能升星** — 已有技能降低冷却 / 提高伤害 / 增加范围
3. **属性提升** — +10% 攻击 / +15% 生命 / +5% 移速 等

---

## 装备系统（局外）

### 槽位
- **武器** — 决定普攻类型和基础攻击属性
- **护甲** — 提供生命值和防御
- **饰品** — 提供特殊效果（击杀回血、经验加成、掉落率等）

### 攻击类型

| 类型 | 普攻形态 | 示例武器 |
|------|---------|---------|
| `melee_arc` | 前方 120° 扇形斩击 | 剑、刀 |
| `melee_circle` | 周身 360° 旋转 | 棍、长枪 |
| `ranged_single` | 指向最近敌人单发 | 飞镖、暗器 |
| `ranged_cone` | 前方扇形多发 | 扇子、暴雨梨花针 |
| `ranged_aoe` | 目标点小范围溅射 | 霹雳弹、雷符 |

### 经济
- 每局结算获得 **铜钱**（用于购买新装备）和 **修为**（用于升级已有装备）
- 铜钱掉落主要由击杀敌人和 Boss 奖励决定

---

## 技能系统

### 主动技能（Q / E / R）
- 按键释放，有冷却
- 方向由 WASD 当前移动方向决定（未移动则朝角色面朝方向）
- 技能形态：直线剑气 / 圆形 AOE / 扇形冲击 / 突进位移

### 被动技能
- 自动触发，无需按键
- 触发类型：间隔触发（如每 15 秒 3 秒无敌）、条件触发（如击杀回复 5% 生命）、常驻（如 +20% 移速）

### 数据定义
每个技能为 `.tres` Resource，包含：id、名称、类型、冷却、效果参数。

---

## 敌人系统

### 基础敌人

| 敌人 | 行为 | 首次出现 |
|------|------|---------|
| 山贼喽啰 | 直线慢速追玩家 | 波次 1 |
| 刺客 | 中速追踪 + 急速冲刺 | 波次 3 |
| 弓箭手 | 保持距离、远程射箭 | 波次 5 |
| 武僧 | 缓慢移动、周身护盾免伤 | 波次 7 |

### 精英（每 5 波）
- 体型 1.5 倍，血量 3 倍
- 至少拥有 1 个特殊技能（金钟罩/剑气/召唤等）
- 击杀掉落高品质奖励

### Boss（每 10 波）
- 唯一在场 + 召唤喽啰
- 血条显示
- 多阶段行为（如半血狂暴）

---

## 波次系统

### 难度公式
- 波次 N 敌人总数 = `5 + N × 3`
- 敌人血量倍率 = `1 + N × 0.1`
- 敌人伤害倍率 = `1 + N × 0.08`
- 同时在场敌人上限 = 30

### 生成规则
- 在地图边缘随机位置生成
- 距玩家 ≥ 300px
- 波次间 8 秒喘息，显示「第 X 波」

## 技术架构

### 场景结构

```
battle.tscn          — 主战斗场景
├── Player           — CharacterBody2D (移动/普攻/技能)
├── EnemySpawner     — Node (波次生成/销毁)
├── ProjectilePool   — Node (弹幕对象池)
├── WaveController   — Node (波次计时/难度)
└── UI               — CanvasLayer (HUD/升级弹窗)

equipment_menu.tscn  — 装备配置场景
main_menu.tscn       — 主菜单
```

### 核心文件

| 文件 | 职责 |
|------|------|
| `player/player.gd` | WASD 移动、自动普攻、技能释放 |
| `enemies/enemy.gd` | 敌人基类（寻路、血量、掉落） |
| `enemies/spawner.gd` | 波次刷怪、敌人生成与上限管理 |
| `skills/skill_manager.gd` | 技能注册、冷却、触发、效果应用 |
| `equipment/equipment_manager.gd` | 装备槽位、属性合计 |
| `wave/wave_controller.gd` | 波次计时、难度递增、波间暂停 |
| `ui/hud.gd` | 血条/经验条/波次/计时 |
| `ui/upgrade_popup.gd` | 升级 3 选 1、暂停/恢复 |
| `data/save_manager.gd` | 铜钱/修为/装备存档 |

### 数据流

```
Enter Battle
  → EquipmentManager 加载玩家装备 → 计算初始属性
  → WaveController 启动波次 1
  → Spawner 生成敌人

Combat Loop
  Player 自动普攻 → 弹幕命中 Enemy → Enemy HP-0
  → Player 获得经验 → 升级触发 → UpgradePopup 暂停战斗
  → 玩家选 1 项 → SkillManager 应用 → 继续

Wave Transition
  本波敌人清零 → WaveController +1 波次
  → Spawner 刷新配置 → 下一波开始

Death
  Player HP=0 → 结算面板
  → SaveManager 写入铜钱/修为
  → 返回主菜单
```

## 测试策略

- 每个管理器类独立单元测试（UTest）
- 战斗核心逻辑用 GUT（Godot Unit Test）覆盖
- 手动测试手感为主（移动+自动攻击+技能释放的流畅度）

## 不在此版本的特性

- 移动端适配
- 多角色
- 存档云端同步
- 成就系统
